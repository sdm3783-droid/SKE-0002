# SKE-0002 "언제오시나?" 전면 리디자인 스펙

## 배경
현재 UI는 인디고/퍼플(#6366f1) 톤의 카드형 대시보드. 사용자가 색감이 별로라며 전면 리디자인을 요청. 레퍼런스로 Agile Analytics(MS Power BI) 스타일 HR 대시보드 2장을 제공받음 — 화이트/라이트그레이 배경, 머스타드 골드 단일 포인트 컬러, 아이콘 헤더 + KPI 타일 그리드, 도넛/바 차트 조합.

## 범위
`index.html` 단일 파일(1933줄) 내 스타일 및 일부 마크업/네비게이션 구조 재작업. 데이터 로직(Firebase 연동, 기록 CRUD)은 변경하지 않음.

## 1. 컬러 팔레트

| 용도 | 색상 |
|---|---|
| 배경 | `#FAFAF9` |
| 카드 배경 | `#FFFFFF`, 보더 `#E8E6E1` |
| 메인 포인트 (액션/활성) | 머스타드 골드 `#D4A017` |
| 서브 포인트 (보조 바/비활성) | 올리브 `#8A6D1B` |
| 텍스트 (제목) | `#1F2937` |
| 텍스트 (보조) | `#6B7280` |
| 상태 - 정상 | 그린 `#22c55e` (기존 유지) |
| 상태 - 지각/이탈 | 레드 `#DC2626` |
| 상태 - 무단결근 | 오렌지 `#EA580C` |
| 로고/강조 그라디언트 | `#D4A017 → #F2B705` (기존 `#6366f1→#8b5cf6` 대체) |

## 2. 내비게이션
- 좌측 고정폭(64px) 아이콘 사이드바 → 상단 가로 탭바로 전환.
- 탭 목록: 대시보드 / 달력 / 통계 / 기록 / 입력 (기존 5개 유지).
- 활성 탭: 골드 밑줄 + 골드 텍스트. 비활성: 그레이 텍스트.
- 클릭 동작: 스크롤 이동 방식 폐기 → 탭당 1개 패널만 표시(show/hide), 나머지 패널은 숨김. `navTo()` 함수를 스크롤 방식에서 패널 토글 방식으로 변경.

## 3. 대시보드 섹션
- 상단에 KPI 요약 타일 4개 신설 (신규 마크업):
  1. 오늘 근무중 인원
  2. 오늘 이탈(지각) 인원
  3. 이번달 총 이탈시간
  4. 이번달 무단결근 건수
- 하단 기존 직원별 상태 카드(`status-card`)는 화이트 카드 + 좌측 상태 보더(정상=그린, 이탈=레드, 결근=오렌지)로 재스타일. 카드 내부 레이아웃/정보는 변경 없음.

## 4. 통계(이탈현황) 섹션
- 기존 세로 막대그래프(`css-bar-chart`) 구조 유지, 막대 색상만 직원별 골드 계열 그라디언트로 교체.
- 요약 KPI 타일 2개 신설: 이번달 총 이탈시간, 무단결근 건수 (대시보드 KPI와 값은 공유 가능, 표시 목적만 다름 — 통계 섹션은 분/시간/일 단위 토글에 연동).

## 5. 달력 섹션
- 구조 변경 없음. 오늘 표시/선택 하이라이트 색상만 인디고 → 골드로 교체.
- 기록 있는 날짜의 미니 아바타 스택 색상도 골드 톤 계열로.

## 6. 기록(테이블) / 입력(폼) 섹션
- 구조 변경 없음.
- 포인트 컬러(버튼, 포커스 보더, 체크박스, 뱃지)만 골드로 교체.
- Primary 버튼(저장 등): 골드 배경 + 진한 텍스트(`#1F2937`).

## 7. 미변경 사항
- Firebase 데이터 연동, 기록 CRUD 로직, 직원 평가(A/B/C/D 등급) 모달 로직 등 기능 로직은 그대로.
- 모바일 반응형 breakpoint 구조는 유지.

## 8. 검증 기준
- 브라우저에서 index.html 열어 5개 탭 전환 시 패널 전환이 올바르게 동작하는지 확인.
- 지각/무단결근 상태 카드가 레드/오렌지로, 정상 상태가 그린으로 표시되는지 확인.
- 기존 기능(입력 폼 제출, 달력 클릭, 직원 평가 모달)이 리디자인 후에도 정상 동작하는지 확인.
